# 基金数据质量分析报告

## 当前问题

### 1. 过度依赖 AkShare 的问题
- **接口不稳定**: 字段名经常变化，导致数据映射失败
- **数据不完整**: 单只基金信息分散在多个接口，需要组合查询
- **Mock 数据泛滥**: 当 AkShare 接口失败时，系统降级到 Mock 数据，导致显示"李四"、"王五"等假数据
- **缺乏数据验证**: 没有验证返回数据的真实性

### 2. 当前数据源对比

| 数据项 | AkShare 可用性 | 当前实现 | 问题 |
|--------|---------------|---------|------|
| 基金名称 | ✅ `fund_name_em` | 使用 | 正常 |
| 基金类型 | ✅ `fund_name_em` | 未使用 | Mock |
| 基金经理 | ✅ `fund_rating_all` | 未使用 | Mock |
| 基金公司 | ✅ `fund_rating_all` | 未使用 | Mock |
| 基金规模 | ⚠️ 需组合查询 | Mock | 不准确 |
| 净值数据 | ✅ `fund_open_fund_daily_em` | 使用 | 正常 |
| 净值历史 | ✅ `fund_open_fund_info_em` | 使用 | 正常 |
| 持仓数据 | ✅ `fund_portfolio_hold_em` | 使用 | 正常 |
| 行业配置 | ✅ `fund_portfolio_industry_allocation_em` | 使用 | 正常 |

### 3. 改进方案

#### 方案 A: 修复 AkShare 数据获取（短期）
- 使用 `fund_name_em` 获取基金名称和类型
- 使用 `fund_rating_all` 获取基金经理和公司
- 组合多个接口获取完整数据
- 增加数据验证层，拒绝明显错误的 Mock 数据

#### 方案 B: 引入多数据源（中期）
- **东方财富**: 基金详情页 API
- **天天基金**: 基金档案数据
- **Wind/Choice**: 专业数据（如有授权）
- **自建数据库**: 定期同步真实数据

#### 方案 C: 数据质量监控（长期）
- 建立数据质量评分机制
- 异常数据自动告警
- 用户反馈数据问题通道

## 建议

1. **立即修复**: 组合使用 `fund_name_em` + `fund_rating_all` 获取真实数据
2. **中期规划**: 引入东方财富/天天基金作为备用数据源
3. **长期建设**: 建立数据质量监控体系

## 关键修复点

### fund_fetcher.py 需要修改：
1. `get_fund_info()` 方法：
   - 调用 `fund_name_em()` 获取名称和类型
   - 调用 `fund_rating_all()` 获取经理和公司
   - 移除 Mock 数据逻辑

2. 增加数据验证：
   - 验证基金经理姓名格式（中文姓名）
   - 验证基金公司名称（知名基金公司列表）
   - 拒绝明显虚假的数据
