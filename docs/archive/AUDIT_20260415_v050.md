# AlphaTerminal v0.5.0 综合审计报告

## 📅 审计时间：2026-04-15

## 一、与专业金融平台的差距评估

### 1.1 现有功能 vs 专业平台（如Wind、东方财富Choice）

| 功能 | AlphaTerminal | 专业平台 | 差距 |
|------|--------------|---------|------|
| 实时行情 | ✅ Sina/AkShare | ✅ 毫秒级 | ⚠️ 延迟3-10秒 |
| K线图表 | ✅ ECharts | ✅ 的专业图表 | ⚠️ 功能完整但性能一般 |
| 技术指标 | ✅ 基础指标 | ✅ 100+指标 | ⚠️ 仅支持10+指标 |
| 财务数据 | ❌ 缺失 | ✅ 完整的财务报表 | 🔴 重大缺失 |
| 资金流 | ❌ 缺失 | ✅ Level2资金流 | 🔴 重大缺失 |
| 消息面 | ✅ 快讯 | ✅ 新闻舆情 | ⚠️ 仅快讯，无分析 |
| 量化工具 | ❌ 缺失 | ✅ 公式编辑器 | 🔴 重大缺失 |
| 风控系统 | ❌ 缺失 | ✅ 实时风控 | 🔴 重大缺失 |
| 投资组合 | ⚠️ 基础 | ✅ 专业组合管理 | ⚠️ 基础功能 |
| WebSocket推送 | ⚠️ 定义但未充分使用 | ✅ 实时推送 | 🔴 未实现实时推送 |

### 1.2 差距总结
- **数据维度**：缺少财务数据、资金流、分析师预期等核心数据
- **实时性**：轮询模式，延迟3-10秒；专业平台使用WebSocket毫秒级推送
- **技术指标**：仅支持基础指标，缺乏专业指标（KDJ、WR、PSY等）
- **用户体验**：UI相对专业，但缺乏个性化定制

## 二、前端UI潜在问题

### 2.1 代码一致性问题

| 问题 | 位置 | 严重程度 |
|------|------|---------|
| 混用fetch和apiFetch | 多个Vue组件 | ⚠️ 中 |
| 某些API缺少错误处理 | AdvancedKlinePanel.vue | ⚠️ 中 |
| quote_detail端点不存在 | FullscreenKline.vue | 🔴 高 |
| 路径不一致 | market.py vs 前端调用 | ⚠️ 中 |

### 2.2 前端API调用问题

```javascript
// 问题1: AdvancedKlinePanel.vue 使用原生fetch
const res = await fetch(`/api/v1/market/quote/${sym}`)

// 问题2: FullscreenKline.vue 调用不存在的端点
const d = await apiFetch(`/api/v1/market/quote_detail/${props.symbol}?_t=${Date.now()}`)
```

### 2.3 路由问题

debug路由注册在sentiment之前，可能导致`/market/sentiment`被`/market/{symbol}`拦截：
```python
app.include_router(debug.router)   # ← 在 sentiment 之前
app.include_router(sentiment.router)  # /market/sentiment 会被 debug 的 /market/{symbol} 拦截
```
