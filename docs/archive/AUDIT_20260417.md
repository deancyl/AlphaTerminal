## 3. 前端UI问题清单

### 🔴 P0 — 严重问题

#### 3.1 ECharts 内存泄漏（部分组件）
**现状**: 16个使用ECharts的组件中，约14个已正确实现 `onBeforeUnmount/onUnmounted` 中的 `dispose()`
**真正有问题的组件**:
- `AdvancedKlinePanel.vue` — **无** `echarts.init` 调用（通过BaseKLineChart代理），但有 `onUnmounted` dispose（间接清理）
- `IndexLineChart.vue` — ✅ 有 `dispose` + `resizeObserver.disconnect()`（顺序正确）
- `FullscreenKline.vue` — ✅ 有双重 `dispose`（`onBeforeUnmount` + `onUnmounted`）
- `BaseKLineChart.vue` — ✅ 有 dispose
- `QuotePanel.vue` — ✅ 有 dispose
- `SentimentGauge.vue` — ✅ 有 dispose（两个实例都清理）
- `FundFlowPanel.vue` — ✅ 有 dispose

**结论**: 子审计报告关于 ECharts 内存泄漏的描述**过于悲观**，大部分组件已正确处理。

#### 3.2 `quote_v2` API 内部错误 ✅ 已修复（本次审计）
**原因**: `fetch_with_fallback` 签名与调用方式不匹配 + `FetcherFactory.get_current_name()` 不存在
**修复**: 重写为直接使用 `FetcherFactory.get_fetcher().get_quote(symbol)`
**验证**: `curl http://192.168.1.50:8002/api/v1/market/quote_v2/sh000001` → ✅ 正常返回

#### 3.3 `fund_flow/industry` 接口500错误 ⚠️ 需关注
**原因**: AkShare `stock_sector_fund_flow_summary` 连接超时（第三方服务不稳定）
**影响**: 板块级资金流数据不可用
**建议**: 添加超时重试 + 缓存降级

---

### 🟡 P1 — 中等问题

#### 3.4 `useMarketStream.js` — 后端无 WebSocket 实现
**现状**: 前端 WebSocket hook 完整，但后端 `websocket.py` 路由存在却无处理函数
**影响**: 实时行情依赖 HTTP 轮询（60秒间隔），无法做到真正实时
**建议**: 实现 WebSocket 端点或改用 SSE

#### 3.5 API 格式基本统一，但存在边界情况
**现状**: 绝大多数 API 返回 `{code, message, data, timestamp}`，`apiFetch` 自动解包 `data`
**边界问题**:
- `/market/quote/{symbol}` 返回数据在 `data` 字段内 ✅
- `/market/history/{symbol}` 返回 `data.history` 数组，`apiFetch` 解包后 `d` 就是数组 ✅
- `FullscreenKline.vue` 中 `d?.history || d || []` 在 `apiFetch` 解包后第一层就是数组，逻辑兼容 ✅

#### 3.6 主题切换 FOUC
**现状**: 主题在 `index.html` 中通过 localStorage 同步设置，有约100ms闪烁风险
**建议**: 在 `<head>` 中内联同步 CSS 变量设置

#### 3.7 Watch 重复触发（IndexLineChart）
**现状**: `watch([props.symbol, props.url, props.indicators], ...)` 中 `props.url` 包含 symbol
**影响**: symbol 变化时双重触发
**建议**: 只 watch `props.symbol`，`props.url` 通过 computed 跟随

---

### ⚪ P2 — 轻微/改进项

#### 3.8 `amplitude=0` 和 `turnover_rate=0`（指数数据）
**原因**: AkShare 日K数据中，指数的 `amount/turnover_rate/amplitude` 字段为0
**影响**: 前端显示时指数幅度为0（用户可能误解）
**建议**: 日K写入时正确填充 amount（从新浪/腾讯接口获取）

#### 3.9 `BacktestDashboard.vue` 和 `AttributionPanel.vue` 功能空壳
**现状**: 组件存在但无实际回测/归因计算逻辑
**建议**: 接入 `backtest.py` 路由的实际计算

#### 3.10 Copilot 依赖模板（无真实LLM时）
**现状**: `copilot.py` 在无 API Key 时使用 `MockLLM` 返回模板内容
**建议**: 添加 `OPENAI_API_KEY` / `MINIMAX_API_KEY` 环境变量提示

## 4. 前后端代码协同问题

### 4.1 API 路径不一致（历史遗留）
**问题**: `/market/fund_flow` 在 `sentiment.py` 中定义为无参数路由，但前端可能在调用时带上了 symbol 参数
```python
# sentiment.py
@router.get("/market/fund_flow")  # 无 {symbol} 路径参数
async def market_fund_flow():      # 全市场资金流，不需要标的
```
**影响**: `curl /market/fund_flow/sh000001` → 404（但这个是正确的设计，全市场数据不需要标的）
**建议**: 前端 `FundFlowPanel.vue` 需确认调用时无 symbol 参数

### 4.2 两套数据源熔断系统并存
**问题**: 
- `backend/app/services/circuit_breaker.py` — 主熔断器
- `backend/app/services/fetchers/circuit_breaker.py` — Fetcher专用熔断器（重复定义）
**建议**: 统一使用 `services/circuit_breaker.py`

### 4.3 `market.py` 行数过多（1806行）
**问题**: 单文件过大，路由和业务逻辑混杂
**建议**: 按功能拆分为 `market_quotes.py`, `market_history.py`, `market_macro.py`
