## 3. 前后端协同问题

### 🔴 高优先级

#### 3.1 Bond 数据源限制（已知问题）
- **现象**: `/bond/curve` 返回 `source: "mock"`，所有字段为空
- **根因**: `akshare.bond_china_yield()` 数据仅到 2021-01-22，AkShare 停止更新
- **影响**: 债券 Dashboard 显示静态 Mock 数据（2021 年数据）
- **修复方向**:
  1. 接入中国债券信息网/其他实时数据源
  2. 或明确标注数据截止日期

#### 3.2 指数 K线 `amplitude`/`turnover_rate`/`amount` 字段为 0
- **现象**: `/market/history` 返回的指数日K线数据中，这三个字段均为 0.0
- **根因**: AkShare `stock_zh_index_daily` 对指数不提供这些字段（代码注释 line 822: "指数无此字段"）
- **影响**: 图表中 amplitude、换手率等副图指标显示为 0
- **修复方向**:
  1. 指数 amplitude 可由 `amplitude = (high - low) / prev_close * 100` 自行计算
  2. turnover_rate 可从 Sina 或其他数据源获取
  3. amount 对于指数没有意义（不适用）

#### 3.3 `/futures/term_structure` 不支持 IF/IC/IM
- **现象**: 调用 `/futures/term_structure?symbol=IF` 返回错误
- **根因**: term_structure 仅支持商品期货，不支持股指期货
- **修复**: `FuturesPanel.vue` 错误处理已修复（`data.error` → `data.code !== 0`）

### ⚠️ 中优先级

#### 3.4 `FundFlowPanel.vue` 直接调用 `apiFetch`
- **现象**: 组件内直接使用 `apiFetch` 而非通过 `useMarketStore`
- **说明**: 功能上正常，但数据流不统一，不利于全局状态管理
- **建议**: 统一数据获取方式

#### 3.5 `market.py` 臃肿（1875 行）
- **现象**: 单个路由文件超过 1800 行
- **风险**: 可维护性差，bug 定位困难
- **建议**: 拆分 market.py 为多个子模块（指数路由、股票路由、实时路由等）

#### 3.6 双 CircuitBreaker 实现
- **现象**: 后端和前端各有一份 CircuitBreaker
- **说明**: 后端在 `app/services/circuit_breaker.py`，前端在 `composables/useDataSourceStatus.js`
- **风险**: 逻辑重复，维护成本高
- **建议**: 统一使用后端熔断状态，前端只做展示
