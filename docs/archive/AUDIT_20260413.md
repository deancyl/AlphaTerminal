## 三、离专业金融平台的差距

### 已实现的功能 ✅
- 多市场指数行情（A股、港股、美股）
- 分钟级K线图表（5/15/30/60分钟）+ 日/周/月K
- 多主题支持（深色/Wind/亮色/全黑4套主题）
- 板块热点分析
- 债券收益率曲线
- 期货主连/期限结构
- 投资组合管理（多账户、持仓、盈亏计算）
- 快讯流
- AI Copilot（支持 LLM API + 本地 WebLLM）
- 绘图工具（标注线、技术图形）

### 缺失的核心功能 ❌ (对比 Wind/同花顺/东方财富)

| 缺失项 | 优先级 | 影响 | 建议 |
|--------|--------|------|------|
| **实时WebSocket推送** | P0 | 用户无法实时接收行情变化 | 实现 ws/market 的数据推送，替代轮询 |
| **个股详情数据源** | P0 | PE/PB/52周高低/资金流全为None | 接入 Sina/AkShare 实时详情 |
| **分时图（1分钟）** | P1 | 目前只支持5分钟起 | Sina不支持1分钟指数，需换数据源 |
| **财务数据（PE/PB/ROE）** | P1 | 投资决策缺乏基本面 | 接入 Eastmoney/AkShare 财务 API |
| **资金流数据** | P1 | 无法判断主力动向 | 接入 DDE/资金流 API |
| **板块成分股** | P2 | 板块分析深度不足 | 接入板块成分股列表 |
| **K线技术指标（MACD/KDJ/BOLL）** | P2 | 技术分析不完整 | 前端计算或后端提供 |
| **回测系统** | P3 | 无法验证策略 | 可选功能 |
| **期权/可转债数据** | P3 | 衍生品缺失 | 可选功能 |
| **预警系统** | P3 | 无法主动提示 | 用户设置价格/指标预警 |

### WebLLM 专项评估
- **优势**: 隐私保护（本地推理），无需 LLM API key
- **问题**: 构建产物 `vendor-webllm.js` 达 5.8MB（压缩后 2.1MB），实际加载需要下载模型（~3GB），加载时间长
- **现状**: 已实现云端/WebLLM 模式切换，移动端自动禁用
- **建议**: WebLLM 仅作为可选模式，默认使用云端 LLM（速度更快、效果更好）

## 四、前端UI潜在问题

### 🔴 严重问题
1. **14 个 API 调用缺少 try/catch**
   - 影响组件: AdvancedKlinePanel, CopilotSidebar, FullscreenKline, FuturesPanel, IndexLineChart, NewsFeed, PortfolioDashboard, SentimentGauge, StockScreener
   - 后果: 网络错误/服务端错误会导致 Vue 组件白屏
   - 修复: 在所有 `fetch`/`apiFetch` 调用外层包裹 try/catch

2. **两套 API 工具不统一**
   - `api.js`（有重试+超时）: 新组件使用
   - `apiCompat.js`（无重试、无超时）: 旧组件（FullscreenKline, IndexLineChart, AdvancedKlinePanel）使用
   - 建议: 统一使用 `api.js`

### 🟡 中等问题
3. **Portfolio 创建错误提示** ✅ 已修复（HTTP 400 → 显示"该账户名称已存在"）
4. **WebSocket 断线重连无限循环** — FuturesPanel 的 WS 连接失败后无限重试，消耗性能
5. **ECharts CDN + npm 按需导入冲突** — 生产构建已 external，但开发环境可能有重复

### 🟢 低风险
6. **API 格式不统一** — 部分返回 `{code, data}`，部分返回 `{code, message, data}`，`data` 层嵌套不一致
7. **TradingView 图表未使用** — 高级图表功能使用 ECharts 而非 TradingView，功能差距较大

## 五、前后端代码协同问题

### 已修复的问题 ✅
- `_normalize_symbol("sz000001")` → `"sh000001"` (平安银行变上证指数)
- `store.positions.reduce()` ref 解包问题
- `fetchAll(store.activePid)` 对象传入问题
- 分钟K线数据源（Eastmoney → Sina）

### 仍存在的协同问题 ⚠️

1. **API 响应格式不统一**
   - `market.py` 用 `{code: 0, message: "success", data: {...}}`
   - 部分接口返回 `{code, data}` 无 message
   - 前端需要做格式兼容 (`apiCompat.js`)

2. **WebSocket 路由不完整**
   - 后端注册 `/ws/market` 路由
   - 前端 `useMarketStream.js` 连接 `/ws/market`（或 `wss://`）
   - 实际 WS 连接一直失败（代理/GnuTLS 不兼容 + 后端未实现推送）

3. **`_SYMBOL_REGISTRY` 只有 5 个指数**
   - 后端 `_SYMBOL_REGISTRY` 硬编码 5 个指数
   - 前端 `symbols.js` 从 `/api/v1/market/symbols` 获取
   - `all_stocks` API 返回 5497 只股票但仅包含 `symbol` 和 `name`
   - 缺少行业、板块、PE/PB 等元数据

4. **scheduler 刷新频率不一致**
   - 日线数据: 30秒（`refresh_today_from_minute`）
   - 实时行情: 60秒（`fetch_all_and_buffer`）
   - 分钟K线: 无 scheduler（仅 API 按需拉取）
   - 快讯: 无 scheduler（仅启动时一次性 + 手动）
