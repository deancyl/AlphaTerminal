## 三、设计系统一致性

### 3.1 颜色系统

**✅ 已标准化**:
- 品牌色: `#0F52BA` (万德蓝)
- 上涨色: `#E63946` (红涨)
- 下跌色: `#1A936F` (绿跌)
- 背景色: `#0D1117` / `#161B22`
- 文字色: 4 级层级

**⚠️ 仍有问题**:
- Chart.js ECharts 配置中仍使用硬编码颜色
- 部分 emoji 使用系统默认颜色
- 某些状态色未统一 (success/warning/info/danger)

### 3.2 圆角系统

**✅ 已标准化**:
- 所有组件: `rounded-sm` (2px)
- 按钮: `rounded-sm` (2px)
- 输入框: `rounded-sm` (2px)
- 卡片: `rounded-sm` (2px)

**⚠️ 例外情况**:
- 圆形按钮/头像: `rounded-full` (正确)
- 部分图表容器可能无边框

### 3.3 阴影系统

**✅ 已标准化**:
- 所有组件: `shadow-sm` (最小阴影)
- 模态框: `shadow-sm` (已降级)

**⚠️ 问题**:
- 万德终端实际上几乎不使用阴影
- 当前阴影在暗色背景下效果不明显
- 可考虑完全移除阴影以更接近万德风格

### 3.4 间距系统

**✅ 已标准化**:
- 基础单位: 4px
- XS: 4px, SM: 8px, MD: 16px, LG: 24px, XL: 32px

**⚠️ 问题**:
- 部分组件使用非标准间距 (如 `p-4` + `gap-3` 混用)
- 表格行高不统一 (24px vs 28px)
- 卡片内边距在移动端过大 (16px → 应 12px)

### 3.5 字体系统

**⚠️ 严重问题**:
- 多处使用 `text-[10px]`，违反万德规范 (最小 11px)
- 移动端更不应小于 12px
- 缺乏字体层级定义 (H1/H2/H3/Body/Caption)
- 中文字体栈未优化 (应加入 `"PingFang SC"`)
