### 差距分析
| 维度 | 当前状态 | 专业平台 | 差距 |
|------|----------|----------|------|
| **数据深度** | 仅有价格/成交量 | 财务/资金流向/大宗交易 | 极大 |
| **图表能力** | 基础K线+MACD | 画线/标注/形态识别 | 大 |
| **风控模块** | 无 | 止损/仓位/VaR | 极大 |

| **键盘操作** | 无 | 快捷键/命令行 | 大 |
| **数据导出** | 无 | Excel/PDF/API | 中 |
| **AI能力** | Mock输出 | 实时分析/建议 | 极大 |
| **移动端** | 无 | 响应式设计 | 大 |

## 二、前端UI潜在问题

### P0 严重问题
1. **NewsFeed.vue**: 固定高度380px，小屏幕溢出
2. **DashboardGrid.vue**: 固定gs-x/gs-w定位，不支持响应式
3. **IndexLineChart.vue**: 分时数据可能有显示bug（见MEMORY.md）
4. **App.vue**: 全局错误处理捕获后直接reload，过于暴力

### P1 中等问题
1. **CopilotSidebar**: 使用mock LLM，无真实AI能力
2. **组件复用**: 大量重复代码，无抽象层
3. **状态管理**: 散落在各组件，无统一store
4. **加载状态**: 无骨架屏，无优雅loading

### P2 改进点
1. 无TypeScript类型检查
2. 无单元测试
3. 无E2E测试
4. 无性能监控
