## 2. 前端UI潜在问题清单

### 🔴 P0 — 严重问题

#### 2.1 ECharts 内存泄漏（多处）
**WHAT**: 多个图表组件在 `onUnmounted` 中未调用 `chart.dispose()`
**WHERE**: 
- `StockScreener.vue` — 完全没有 dispose
- `IndexLineChart.vue` — 有 dispose 但 `resizeObserver?.disconnect()` 顺序可疑
- `FullscreenKline.vue` — 有 `_fullscreenRO?.disconnect()` 但未在 `onUnmounted` 中调用

**WHY**: ECharts 实例不释放 → 内存持续增长 → 浏览器卡顿/崩溃

**FIX**: 统一在 `onBeforeUnmount` 中按顺序清理：
```js
onBeforeUnmount(() => {
  resizeObserver?.disconnect()
  chartInstance?.dispose()
  chartInstance = null
})
```

#### 2.2 `apiFetch` 响应结构处理不一致
**WHAT**: `api.js` 的 `apiFetch` 解包 `{code, message, data}`，但某些组件直接访问根级字段
**WHERE**: `FullscreenKline.vue` 第234行：`const historyArray = d?.history || d || []`
**WHY**: 如果后端返回 `{code:0, data: {history: [...]}}`，`d.history` 是 undefined，应走 `d.data.history`
**FIX**: 统一所有 API 调用方使用 `apiFetch` 返回的已解包 `data` 字段

#### 2.3 前端版本号不统一
**WHAT**: `package.json` 版本 0.5.45，后端 `/api/v1/system/version` 报告 frontend 为 0.4.133
**WHERE**: `backend/app/routers/system.py` 返回硬编码 `frontend: "0.4.133"`
**WHY**: 版本号混乱，用户无法判断实际部署版本
**FIX**: 后端从 `frontend/package.json` 动态读取版本

---

### 🟡 P1 — 中等问题

#### 2.4 `IndexLineChart` watch 重复触发
**WHERE**: `IndexLineChart.vue` 第 `watch(() => [props.symbol, props.url, props.indicators], () => { fetchAndRender() }, { deep: true })`
**WHY**: `props.url` 包含 symbol，symbol 变化时两个 watcher 都触发 → 双重请求
**FIX**: 只 watch `props.symbol`，`props.url` 变化通过 computed 跟随

#### 2.5 主题切换闪烁（FOUC）
**WHERE**: `App.vue` + `index.html` 中主题在 Vue 加载前通过 `localStorage` 手动设置
**WHY**: 主题通过 JS 同步，有约 100ms 闪烁风险
**FIX**: 在 `<head>` 中使用同步 CSS 变量注入，无 JS 依赖

#### 2.6 `useMarketStream` — 后端无 WebSocket 端点
**WHAT**: 前端 WebSocket hook 完整实现，但后端没有 `/ws/market` 端点
**WHERE**: `useMarketStream.js` 连接 `/ws/market`，但 `main.py` 中无对应 WebSocket 路由
**WHY**: 实时行情依赖 HTTP 轮询，无法获得真正的实时推送
**FIX**: 在 `main.py` 添加 WebSocket 支持或改用 SSE

#### 2.7 全局状态潜在竞态
**WHERE**: `useMarketStore.js` 的模块级单例 `currentSymbol` 等
**WHY**: 如果多实例组件同时修改，可能出现不一致
**FIX**: 考虑迁移到 Pinia（已在 `stores/drawing.js` 使用）

#### 2.8 骨架屏不一致

## 3. 前后端代码协同问题

### 3.1 `api.js` 与 `apiCompat.js` 两套并存
**WHAT**: 项目同时存在 `api.js` 和 `apiCompat.js`，职责边界不清晰
**WHERE**: `frontend/src/utils/`
**FIX**: 合并为单一 `api.js`，统一错误处理和响应解包

### 3.2 `market.py` 多处不一致的数据结构
**WHAT**: 不同 API 端点返回格式不统一
- `/market/quote/{symbol}` → `{symbol, price, change, ...}` (无 `data` 包装)
- `/market/history/{symbol}` → `{code, data: {history: [...]}}`
- `/market/all_stocks_lite` → `{data: {stocks: [...]}}`

**FIX**: 统一响应格式为 `{code, data, message}`，在 `apiFetch` 中完全解包

### 3.3 `volume` 字段语义混淆
**WHAT**: 
- 指数的 `volume` 在 `market_data_realtime` 中是**成交量（股数）**
- 某些接口返回 `volume` 是**成交额（元）**
- `FundFlowPanel` 展示时未明确标注单位

**FIX**: 统一字段命名，`volume`=成交量，`amount`=成交额

### 3.4 数据库 Schema 与 API 返回字段不匹配
**WHAT**: 
- `market_data_daily` 有 `amount/turnover_rate/amplitude` 列
- 但 `get_daily_history` SELECT * 返回这些列
- AkShare 写入时 `amount=0.0`（指数日K无成交额）

**FIX**: 日K写入时正确填充 `amount`（从前端数据源获取）
