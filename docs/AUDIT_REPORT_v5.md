## 1. 与专业金融平台的差距评估

### 差距矩阵 (对比 Wind/同花顺/东方财富)

| 维度 | 专业平台 | AlphaTerminal | 差距 |
|------|---------|---------------|------|
| **数据实时性** | Level-2 逐笔/3秒刷新 | 60秒轮询 | 🔴 极大 |
| **K线功能** | 复权/画线/多周期联动 | 基础K线+8指标 | 🟡 中等 |
| **新闻舆情** | NLP情感分析/热点追踪 | 基础快讯展示 | 🟡 中等 |
| **Copilot** | 研报生成/策略回测 | 基础问答+WebLLM | 🟡 中等 |
| **组合分析** | 风险分析/归因/回撤 | 简单持仓盈亏 | 🔴 极大 |
| **全市场覆盖** | 5000+股票实时 | 50只重点股 | 🔴 极大 |
| **技术指标** | 100+指标库 | 8个副图指标 | 🟡 中等 |
| **画线工具** | 专业套装 | 11种基础画线 | 🟢 较小 |

### 核心差距总结
1. **实时性不足**: 60秒轮询 vs 专业平台3秒推送
2. **数据覆盖窄**: 仅50只重点股 vs 全市场5000+
3. **分析能力弱**: 缺少风险分析、归因分析、回测
4. **移动端体验差**: WebLLM内存占用导致崩溃

## 2. 前端UI潜在问题

### 🔴 P0 - 立即修复

**2.1 K线数据不自动刷新**
- **文件**: `FullscreenKline.vue`
- **问题**: `onMounted` 只调用一次 `fetchData()`，无定时刷新
- **影响**: 用户必须手动切换周期才能看到新数据
- **修复**: 添加 `setInterval` 每30秒刷新，或WebSocket推送

**2.2 WebLLM 移动端崩溃**
- **文件**: `CopilotSidebar.vue` line 211-225
- **问题**: 尝试顺序加载3个模型 (Llama-3.1-8B, Qwen2.5-7B, Phi-3.5)
- **影响**: 移动端浏览器内存不足崩溃
- **修复**: 
  - 仅尝试加载1个轻量级模型
  - 添加内存检测，低内存设备禁用WebLLM
  - 默认使用云端模式

### 🟠 P1 - 高优先级

**2.3 StockScreener 名称宽度过宽**
- **文件**: `StockScreener.vue`
- **问题**: 股票名称列无最大宽度限制，长名称撑破布局
- **修复**: 添加 `max-width` + `truncate` + `title` 提示

**2.4 Portfolio 功能不完整**
- **文件**: `PortfolioDashboard.vue`
- **现状**: 仅有基础持仓展示和简单盈亏计算
- **缺失**: 
  - 风险指标 (VaR, 夏普比率)
  - 归因分析 (行业/个股贡献)
  - 回撤分析 (最大回撤、回撤天数)
  - 交易记录和成本分析

**2.5 无骨架屏/加载状态**
- **问题**: 多个组件加载时显示空白，用户体验差
- **影响**: 用户不知道数据正在加载

### 🟡 P2 - 中优先级

**2.6 响应式布局问题**
- **问题**: 移动端 `<768px` 时多个组件布局错乱
- **文件**: `DashboardGrid.vue`, `FullscreenKline.vue`

**2.7 错误处理不完善**
- **问题**: API失败时显示空白或报错，无降级UI
- **文件**: 多个组件的 `catch` 块仅 `console.error`

## 3. 前后端代码协同问题

### 🔴 P0

**3.1 数据刷新机制不一致**
- **前端**: 依赖 `onMounted` 一次性加载
- **后端**: 60秒轮询刷新
- **问题**: 前端数据比后端滞后，用户看到旧数据
- **修复**: 前端添加定时刷新，或WebSocket实时推送

**3.2 API响应格式仍有不一致**
- **问题**: 部分旧端点可能仍返回裸数据
- **状态**: 大部分已修复为 `{code, data, message, timestamp}`

### 🟠 P1

**3.3 全市场个股数据覆盖不足**
- **后端**: `SpotCache` 仅缓存50只重点股
- **前端**: `StockScreener` 从这50只中筛选
- **问题**: 用户无法搜索/查看全市场5000+股票
- **修复**: 接入 `akshare stock_zh_a_spot_em()` 全量数据

**3.4 缺少数据流架构**
- **现状**: 轮询式数据获取
- **问题**: 高延迟、高服务器负载
- **建议**: 迁移至 WebSocket/SSE 推送架构
