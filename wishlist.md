## 🎯 当前开发重点（基于 Karpathy 准则）

### 开发原则

1. **Think Before Coding**: 每个任务开始前明确假设和验收标准
2. **Simplicity First**: 最小代码解决问题，不添加未要求的功能
3. **Surgical Changes**: 只修改必要的代码，不触碰无关文件
4. **Goal-Driven Execution**: 每个任务都有可验证的成功标准

### 当前阶段目标

**Phase 2.1: 技术债清理 + 核心体验（Week 1-4）- v0.6.x 阶段**
- 消除所有 Critical 安全漏洞
- 实现 Wind 风格键盘快捷键
- 搭建 EDB 宏观经济基础框架

**说明**: 如无特殊说明，所有开发任务均属于 v0.6.x 版本迭代，不提升大版本号。

## 🔧 技术架构演进

### 当前架构（单体应用）
- **后端**: FastAPI + SQLite + APScheduler
- **前端**: Vue3 + Vite + TailwindCSS + Pinia
- **数据源**: AkShare + Sina + Eastmoney
- **部署**: 单节点本地运行

### Phase 2 架构优化（高并发支持）
- **连接池**: SQLAlchemy连接池替代每次新建连接
- **缓存层**: Redis集群（行情缓存、会话存储）
- **CDN**: 静态资源加速
- **WAL模式**: SQLite并发读写优化（已完成）

### Phase 3 架构升级（微服务化）
- **API网关**: Kong/Nginx（负载均衡、限流、鉴权）
- **服务拆分**:
  - 行情服务（Market Service）- 高频数据推送
  - 计算服务（Calculation Service）- 指标计算、回测
  - 数据服务（Data Service）- EDB、财务数据
  - 通知服务（Notification Service）- WebSocket、推送
- **消息队列**: RabbitMQ/Kafka（异步任务、事件驱动）
- **边缘计算**: 全球金融节点部署（香港、新加坡、东京、伦敦、纽约）

### Phase 4 前沿技术
- **WebAssembly**: 前端高性能计算（技术指标实时计算）
- **时序数据库**: InfluxDB/TimescaleDB（高频行情存储）
- **图数据库**: Neo4j（企业图谱、关系网络）
- **向量数据库**: Milvus/Pinecone（语义搜索、研报相似度）
- **搜索引擎**: Elasticsearch（全文检索、智能联想）

## 📐 系统架构总则

基于新PRD要求，系统架构必须满足：

### 1. 性能要求
- **延迟**: 实时行情 < 500ms，F9页面 < 2s，EDB查询 < 5s
- **并发**: 支持1000+自选股同时监控，10000+并发连接
- **可用性**: 7×24×365，可用性 ≥ 99.9%

### 2. 数据架构
- **实时流**: WebSocket + 边缘计算节点
- **批处理**: APScheduler + 分布式爬虫集群
- **存储分层**:
  - 热数据（实时行情）: Redis
  - 温数据（日K/财务）: PostgreSQL/TimescaleDB
  - 冷数据（历史宏观）: 对象存储

### 3. 前端架构
- **渲染优化**: 虚拟滚动、虚拟DOM复用、WebAssembly计算
- **状态管理**: Pinia + 局部状态（避免全局污染）
- **模块化**: GridStack灵活布局、组件懒加载
- **交互**: 全局快捷键、键盘精灵、语义搜索

### 4. 安全架构
- **传输**: TLS 1.3 全链路加密
- **鉴权**: JWT + OAuth2.0
- **风控**: API限流、SQL注入防护、XSS过滤
- **审计**: 操作日志全量记录

## 🔧 技术架构演进

### 当前架构（单体应用）
- **后端**: FastAPI + SQLite + APScheduler
- **前端**: Vue3 + Vite + TailwindCSS + Pinia
- **数据源**: AkShare + Sina + Eastmoney
- **部署**: 单节点本地运行

### Phase 2 架构优化（高并发支持）
- **连接池**: SQLAlchemy连接池替代每次新建连接
- **缓存层**: Redis集群（行情缓存、会话存储）
- **CDN**: 静态资源加速
- **WAL模式**: SQLite并发读写优化（已完成）

### Phase 3 架构升级（微服务化）
- **API网关**: Kong/Nginx（负载均衡、限流、鉴权）
- **服务拆分**:
  - 行情服务（Market Service）- 高频数据推送
  - 计算服务（Calculation Service）- 指标计算、回测
  - 数据服务（Data Service）- EDB、财务数据
  - 通知服务（Notification Service）- WebSocket、推送
- **消息队列**: RabbitMQ/Kafka（异步任务、事件驱动）
- **边缘计算**: 全球金融节点部署（香港、新加坡、东京、伦敦、纽约）

### Phase 4 前沿技术
- **WebAssembly**: 前端高性能计算（技术指标实时计算）
- **时序数据库**: InfluxDB/TimescaleDB（高频行情存储）
- **图数据库**: Neo4j（企业图谱、关系网络）
- **向量数据库**: Milvus/Pinecone（语义搜索、研报相似度）
- **搜索引擎**: Elasticsearch（全文检索、智能联想）

## 📐 系统架构总则

基于新PRD要求，系统架构必须满足：

### 1. 性能要求
- **延迟**: 实时行情 < 500ms，F9页面 < 2s，EDB查询 < 5s
- **并发**: 支持1000+自选股同时监控，10000+并发连接
- **可用性**: 7×24×365，可用性 ≥ 99.9%

### 2. 数据架构
- **实时流**: WebSocket + 边缘计算节点
- **批处理**: APScheduler + 分布式爬虫集群
- **存储分层**:
  - 热数据（实时行情）: Redis
  - 温数据（日K/财务）: PostgreSQL/TimescaleDB
  - 冷数据（历史宏观）: 对象存储

### 3. 前端架构
- **渲染优化**: 虚拟滚动、虚拟DOM复用、WebAssembly计算
- **状态管理**: Pinia + 局部状态（避免全局污染）
- **模块化**: GridStack灵活布局、组件懒加载
- **交互**: 全局快捷键、键盘精灵、语义搜索

### 4. 安全架构
- **传输**: TLS 1.3 全链路加密
- **鉴权**: JWT + OAuth2.0
- **风控**: API限流、SQL注入防护、XSS过滤
- **审计**: 操作日志全量记录
