## 2. 数据库查询优化

### 2.1 发现的优化点

| 问题 | 位置 | 影响 |
|------|------|------|
| SELECT * 查询 (8处) | database.py | 获取不必要字段，增加IO |
| 无连接池管理 | database.py | SQLite顺序执行，高并发下排队 |
| 无查询超时 | database.py | 慢查询会阻塞其他操作 |
| WAL模式由路径决定 | database.py L31 | 某些环境下禁用WAL影响并发 |
| busytimeout仅30s | database.py L39 | 高并发场景可能超时 |

### 2.2 已实施的优化

- WAL 模式配置 (已有，用于支持并发读)
- Busy timeout 配置 (已有，30秒)
- 写操作使用独立队列 (db_writer.py)

### 2.3 建议的进一步优化

| 优化项 | 优先级 | 预期收益 |
|--------|--------|----------|
| 添加数据库连接池 | 高 | 减少连接开销 |
| 优化 SELECT * → 指定列 | 中 | 减少IO 30%+ |
| 添加查询超时机制 | 中 | 防止慢查询阻塞 |
| 关键查询添加索引 | 中 | 查询性能提升10x+ |
