## Phase 5.2: LLM 动态配置穿透测试 ✅ 完成

### 测试目标
验证 Admin 面板的 LLM 配置是否能**真正穿透**到 Copilot 调用链路，杜绝"管理面板存盘成功，但 AI 调用依然读取旧配置"的断层现象。

### 测试步骤

#### 1. 脏数据注入
```bash
POST /api/v1/admin/settings/llm
{
  "provider": "deepseek",
  "api_key": "sk-test-fake-key-12345",
  "base_url": "https://api.fake-deepseek-test-url.com/v1",
  "model": "deepseek-chat"
}
```
✅ 保存成功，`has_db_config: true`

#### 2. Copilot 链路穿透追踪
```bash
POST /api/v1/chat
{
  "prompt": "测试配置穿透",
  "provider": "deepseek"
}
```

**后端日志证据：**
```
[app.routers.copilot] WARNING: [DeepSeek] 调用配置：
  base_url=https://api.fake-deepseek-test-url.com/v1
  model=deepseek-chat
  url=https://api.fake-deepseek-test-url.com/v1/chat/completions
  api_key_masked=sk-test-fa...
[app.routers.copilot] ERROR: [DeepSeek] [Errno -2] Name or service not known
```

**测试结果：**
- ✅ 后端 HTTP Client 确实尝试连接 `https://api.fake-deepseek-test-url.com/v1`
- ✅ **不是** `.env` 中的旧地址
- ✅ **不是** 官方默认地址 `https://api.deepseek.com`
- ✅ 配置链路完全打通：Admin 面板 → SQLite 数据库 → `_get_llm_config()` → `_call_deepseek()` → httpx 请求

#### 3. 容错表现
- ✅ 异常被正确捕获（`[Errno -2] Name or service not known`）
- ✅ 前端收到 SSE 错误消息：`{"error": "DeepSeek API 调用失败：[Errno -2] Name or service not known"}`
- ✅ 无 HTTP 500，无白屏

### 代码修复

**问题：**
```python

cfg = _get_llm_config(provider)
```

## 关键结论

**配置穿透链路已验证：**
```
AdminDashboard.vue (前端 UI)
    ↓ POST /api/v1/admin/settings/llm
backend/app/routers/admin.py (保存配置)
    ↓ INSERT/REPLACE INTO admin_config
SQLite database.db (持久化存储)
    ↓ _get_llm_config("deepseek")
backend/app/routers/copilot.py (读取配置)
    ↓ base_url, api_key, model
_call_deepseek() → httpx.AsyncClient
    ↓ POST {base_url}/chat/completions
LLM Provider (DeepSeek / 假 URL 测试)
```

**优先级顺序：** 数据库配置 > `.env` 环境变量 > 默认值
